# Contributing

Thanks for helping improve Mini Sudoku Go!

## Getting started

```bash
go run ./cmd/mini-sudoku-go
```

## Code style

- Use `gofmt` on all Go files.
- Prefer small, focused functions with doc comments.
- Keep UI changes readable and consistent with existing style.

## Tests

There are no automated tests yet. If you add logic-heavy features, consider adding tests.

## Pull requests

- Describe the change and why it helps.
- Include screenshots or clips for UI changes.
- Keep PRs focused and avoid unrelated refactors.

## Reporting issues

Use the issue templates for bugs and feature requests.
